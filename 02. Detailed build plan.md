## 2. Purpose  
This guide helps sales employees complete common email and follow-up tasks in Classic Outlook for Windows.  
It combines:  
* Detailed procedures  
* Practical decision guidance  
* Screenshot-supported instructions  
* Copilot examples  
* Platform-difference notes  
* Troubleshooting  
* Safe-use reminders  
* Support information  
The guide should help employees complete a task without needing to read the entire document.  
  
## 3. Performance goal  
After using this guide, employees can:  
1. Decide what an email needs.  
2. Make follow-up visible.  
3. Organize messages without losing work.  
4. find messages, people and attachments.  
5. Use Classic Outlook tools consistently.  
6. Use Copilot to summarize, draft and improve messages.  
7. Review Copilot output before sending.  
8. Recognize when a step differs on another Outlook platform.  
9. Find the right support path when a feature is unavailable.  
  
## 4. Core workflow  
Use the same workflow throughout the guide:  
## Email → Decide → Make visible → Follow through  
The guide should repeatedly reinforce these questions:  
1. **Does this need action from me?**  
2. **What is the next step?**  
3. **Where will I make it visible?**  
Use this framework for every inbox-management procedure:  

| Door | Use when | Common action |
| ---------------------- | ---------------------------------- | ------------------------------------------- |
| Respond now | A quick reply resolves the message | Reply or reply all |
| Do the task | The task can be completed quickly | Complete it, then file the email |
| Make follow-up visible | More work or waiting is required | Flag, categorize, calendar or create a task |
| Let it go | No action is required | Archive, file or delete |

Every screenshot placeholder should follow this format:  
**[Insert screenshot here: Exact screen, command and user action.]**

**Callouts:**  
1. [Callout text]

3. [Callout text]  
**Alt text:** One concise sentence explaining the screenshot and its purpose.  
**Snagit notes:**  
* Capture at 100% display scaling when possible.  
* Use Bluejay #41B6E6 callouts.  
* Use no more than three callouts.  
* Use 2 pt strokes.  
* Crop tightly to the needed area.  
* Blur names, email addresses, account data, dates and customer information.  
* Replace live data with realistic dummy data when practical.  
* Do not rely on color alone to communicate meaning.  
## Screenshot sequence rule  
Use screenshots only when they reduce ambiguity.  
Recommended pattern:  
1. Starting location  
2. Command or setting  
3. Result  
Avoid showing a screenshot for every click.
