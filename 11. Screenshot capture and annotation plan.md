## Screenshot capture and annotation plan  
## Take Control of Your Inbox: Classic Outlook for Windows  
## 1. Purpose  
Use this plan to capture, annotate, review and place every screenshot needed for the Classic Outlook guide.  
The screenshot set should:  
* Match the written procedures  
* Use realistic dummy data  
* Show only the interface needed for the task  
* Make commands easy to locate  
* Avoid exposing customer, employee or business information  
* Remain readable in Word and the published PDF  
* Support future updates without rebuilding the full guide  
  
## 2. Capture environment  
## Required setup  
Capture screenshots using:  
* Company-managed Windows laptop  
* Current supported Classic Outlook for Windows

* Default or near-default ribbon layout

## 4. Screenshot visual standard  
## Image quality  
* Capture in PNG format  
* Do not use JPEG for interface screenshots  
* Capture at native resolution  
* Do not enlarge low-resolution captures  
* Keep text sharp  
* Use screenshots at 100% size when possible  
* Maintain consistent window dimensions throughout the guide  
## Cropping  
Crop tightly enough to emphasize the task while retaining enough context to identify:  
* The Outlook application  
* The relevant ribbon tab  
* The selected message or folder  
* The command being used  
Avoid:  
* Full desktop captures when only a dialog is needed  
* Large areas of unused white space  
* Cropping out the name of the active ribbon tab  
* Cropping out the selected mailbox when mailbox context matters  
## Screenshot frame  
In Word:  
* Place screenshots inside a light gray frame  
* Use a 0.5–1 pt border  
* Do not use decorative shadows  
* Keep screenshots aligned to the page grid  
* Use the same width for comparable screenshots  
* Place captions directly below the image  
  
## 5. Annotation standard  
## Callout appearance  
Use:  
* Bluejay #41B6E6  
* White callout text  
* 2 pt outline or leader line  
* Rounded rectangular labels  
* Numbered circles when order matters  
* Arrows only when they clarify location or movement  
## Callout limits  
* Maximum three callouts per screenshot  
* One idea per callout  
* Keep labels to three to six words  
* Do not cover interface text  
* Do not point through another callout  
* Do not use color as the only indicator  
## Recommended labels  
Use action-focused labels such as:  
* Select Reply  
* Open Follow Up  
* Choose Custom  
* Confirm From address  
* Select Draft with Copilot  
Avoid vague labels such as:  
* Click here  
* This button  
* Important  
* Look here  
## Highlighting  
Use one of these methods:  
1. Bluejay outline around the command  
2. Numbered Bluejay callout  
3. Subtle dimming of unrelated screen areas  
Do not combine all three unless the screen is unusually complex.  
  
## 6. Alt text standard  
Alt text should explain:  
* What screen is shown  
* Which control is emphasized  
* Why the screenshot matters  
## Formula  
**[Application or screen] showing [main control] used to [purpose].**  
## Example  
Classic Outlook message list showing the Follow Up menu used to assign a due date to an email.  
## Avoid  
* “Screenshot of Outlook”  
* File names  
* Decorative descriptions  
* Repeating the full nearby procedure  
* Starting with “Image of”  
Screenshots that are fully explained by adjacent text may be marked decorative only when they add no additional instructional meaning.

## 10. Screenshots not recommended  
Do not create screenshots for:  
* Every basic Reply or Send action  
* Every checkbox in a long wizard  
* Every item in a troubleshooting table  
* Generic reminders  
* The four-door triage framework  
* Platform-comparison tables  
* Final review checklists  
* Support contact information  
These sections are clearer as text, diagrams or tables.  
  
## 11. Screenshot placement rules  
## Place the screenshot  
* Immediately after the step that introduces the screen  
* Before detailed interpretation when the image provides context  
* On the same page as its primary procedure when possible  
* Beside short instructions only when the layout remains readable  
## Do not  
* Place screenshots several pages after the related task  
* Split a screenshot and its caption across pages  
* Allow screenshots to break numbered-step flow  
* Use screenshots as page backgrounds  
* Place text directly over screenshots in Word

## 13. Capture workflow  
## Before capture  
1. Confirm the Outlook version.  
2. Load the dummy mailbox.  
3. Reset Outlook to the approved layout.  
4. Close unrelated applications.  
5. Clear notifications.  
6. Confirm no real data is visible.  
7. Set the correct screen resolution.  
8. Open Snagit or the approved capture tool.  
9. Review the inventory.  
10. Capture one section at a time.  
## During capture  
1. Capture the unannotated source.  
2. Confirm the text is sharp.  
3. Confirm the correct command is visible.  
4. Review the full image for sensitive data.  
5. Save using the naming convention.  
6. Record the capture in the tracker.  
7. Create the annotated copy.  
8. Write the alt text.  
9. Place a temporary copy in the Word draft.  
10. Continue to the next screenshot.  
## After capture  
1. Compare each image with the written procedure.  
2. Confirm command names match.  
3. Check that callouts do not block controls.  
4. Check readability at the final printed size.  
5. Verify alt text.  
6. Confirm source and annotated files are stored separately.  
7. Mark the screenshot Ready for review.

## 15. Review checklist  
## Technical accuracy  
* Screenshot is from Classic Outlook  
* Correct mailbox type is shown  
* Ribbon tab is correct  
* Command name matches the guide  
* Selected item matches the procedure  
* No New Outlook screens are included  
* Copilot is shown only in a supported mailbox  
* Shared mailbox steps use the correct shared-mailbox workflow  
## Visual quality  
* Image is sharp  
* Crop is intentional  
* Interface text is readable  
* Callouts are aligned  
* No callout covers the command  
* Maximum of three callouts  
* Bluejay is used consistently  
* The screenshot fits the Word layout  
## Privacy and security  
* All names are fictional  
* All addresses are fictional  
* No live customer data appears  
* No real attachments appear  
* No account or member numbers appear  
* No verification codes appear  
* No real mailbox permissions appear  
* No confidential interface elements appear  
## Accessibility  
* Alt text explains the screenshot’s instructional purpose  
* Color is not the only way meaning is communicated  
* Numbered callouts follow a logical order  
* Caption is included  
* Screenshot remains readable at 200% zoom  
* Important text is also included in the written procedure  
  
## 16. Maintenance standard  
Review screenshots when:  
* Microsoft changes the ribbon  
* Command names change  
* Copilot controls move  
* Shared mailbox setup changes  
* Microsoft 365 Apps changes update channel  
* The guide receives an outdated-content report  
* Screenshots no longer match employee devices  
* A task generates repeated support questions  
## Update rule  
When replacing a screenshot:  
1. Preserve the original in the Retired folder.  
2. Capture the new source image.  
3. Increase the screenshot version.  
4. Update the annotation.  
5. Review the alt text.  
6. Replace the image in Word.  
7. Recheck pagination.  
8. Re-export the PDF.  
9. Update the guide version when the change is material.  
10. Record the change in the version log.

* Unannotated PNG source files  
* Annotated PNG files  
* Screenshot tracking workbook or table  
* Alt-text inventory  
* Version-evidence screenshot  
* Retired screenshot folder  
* Screenshot placement map  
* Final approved images embedded in the Word guide  
This plan uses the project standard of Bluejay annotations, no more than three callouts, 2 pt strokes, dummy data and concise alt text. Current Microsoft guidance continues to show Classic Outlook’s Focused Inbox under the View tab and describes Search Folders as virtual views that do not move or duplicate messages.
