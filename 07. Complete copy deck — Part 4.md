## Page 28 — Create folders  
## Create folders  
## Use this when  
Store completed or reference messages where you can find them later.  
## Before creating a folder  
Ask:  
* Will I use this folder regularly?  
* Does another folder already serve the same purpose?  
* Is the folder name clear?  
* Will the folder help me retrieve information?  
* Am I using the folder to hide unfinished work?  
## Create a folder  
1. Locate the mailbox or folder where the new folder should appear.  
2. Right-click the mailbox name or parent folder.  
3. Select **New Folder**.  
4. Enter a short, clear name.  
5. Press **Enter**.  
## Create a subfolder  
1. Right-click the folder that should contain the subfolder.  
2. Select **New Folder**.  
3. Enter the subfolder name.  
4. Press **Enter**.  
## Rename a folder  
1. Right-click the folder.  
2. Select **Rename Folder**.  
3. Enter the new name.  
4. Press **Enter**.  
Some default Outlook folders cannot be renamed.  
## Delete an unused folder  
1. Confirm that the folder does not contain messages you need.  
2. Right-click the folder.  
3. Select **Delete Folder**.  
4. Confirm the action.  
## Suggested simple folder structure  
* Inbox  
* Action support  
* Customer reference  
* Producer reference  
* Internal reference  
* Completed projects  
* Archive  
Use only the folders that support your work.

* Do not create several folders with nearly identical names.  
* Do not move unresolved messages unless the next action is visible elsewhere.  
* Do not store a message in several folders unnecessarily.

## Page 29 — Move or copy messages  
## Move or copy messages  
## Use this when  
Place completed or reference messages in the appropriate folder.  
## Move a message using the ribbon  
1. Select one or more messages.  
2. On the **Home** tab, select **Move**.  
3. Choose a recently used folder.  
If the folder is not listed:  
1. Select **Other Folder**.  
2. Select the destination folder.  
3. Select **OK**.  
The message leaves the current folder and appears in the selected destination.  
## Drag a message to a folder  
1. Select the message.  
2. Hold the mouse button.  
3. Drag the message to the destination folder.  
4. Release the mouse button.  
5. Confirm that the message appears in the folder.  
## Copy a message  
Use a copy only when the message must remain in its current folder and also appear elsewhere.  
1. Select the message.  
2. Open the **Move** menu.  
3. Select **Copy to Folder**.  
4. Select the destination.  
5. Select **OK**.  
## Move several messages  
1. Hold **Ctrl** and select individual messages.  
Or:  
1. Select the first message.  
2. Hold **Shift**.  
3. Select the last message in a continuous range.  
Then:  
1. Select **Move**.  
2. Choose the destination folder.

## Page 30 — Archive or delete messages  
## Archive or delete a message  
## Use this when  
Remove a message from the inbox after deciding that it no longer needs attention.  
## Archive a message  
Use Archive when you want to keep a message but remove it from the inbox.  
1. Select one or more messages.  
2. Select **Archive** on the ribbon.  
Or:  
1. Right-click the message.  
2. Select **Archive**.  
3. Confirm that the message leaves the inbox.  
4. Open the **Archive** folder when you need to retrieve it.  
## Delete a message  
Use Delete when the message no longer needs to be retained in your active folders.  
1. Select one or more messages.  
2. Select **Delete**.  
Or:  
1. Press the **Delete** key.  
2. Confirm that the message appears in **Deleted Items**.  
## Restore a deleted message  
1. Open **Deleted Items**.  
2. Select the message.  
3. Select **Move**.  
4. Choose the correct folder.  
You may also drag the message back to the appropriate folder.  
## Recover a message no longer in Deleted Items  
When the organization’s retention settings permit recovery:  
1. Open **Deleted Items**.  
2. Select **Recover Deleted Items from Server** or the available recovery command.  
3. Select the item.  
4. Select **Restore Selected Items**.  
5. Select **OK**.  
Availability and retention periods depend on organization settings.  
## Archive or delete?  

| Action | Use when | Result |
| ------- | -------------------------------------------- | ------------------------------ |
| Archive | You want to keep the message | Message moves to Archive |
| Delete | The message no longer belongs in active mail | Message moves to Deleted Items |
  
****Important note****  
The standard **Archive** folder is different from an **Online Archive** or **In-Place Archive** mailbox that may be enabled by the organization.  
The standard **Archive** folder is different from an **Online Archive** or **In-Place Archive** mailbox that may be enabled by the organization.  
## What to remember  
Archive completed information. Do not archive unresolved work unless the next action is already tracked.  
## Avoid this  
* Do not archive a message to avoid making a decision.  
* Do not permanently delete business records outside approved practices.  
* Do not assume Deleted Items is permanent storage.  
* Do not confuse the Archive folder with an Online Archive mailbox.  
## Platform difference  
Archive and recovery commands differ across Outlook platforms and may depend on organizational retention settings.

## Page 31 — Turn Focused Inbox on or off  
## Use Focused Inbox  
## Use this when  
Separate messages Outlook identifies as more important from messages placed in the Other view.  
Focused Inbox creates two views:  
* **Focused**  
* **Other**  
It does not move messages into separate folders.  
## Turn Focused Inbox on  
1. Open the **View** tab.  
2. Select **Show Focused Inbox**.  
3. Confirm that the **Focused** and **Other** tabs appear above the message list.  
## Turn Focused Inbox off  
1. Open the **View** tab.  
2. Clear **Show Focused Inbox**.  
3. Confirm that the message list returns to one combined view.  
## Switch between views  
1. Select **Focused** to review messages Outlook considers more relevant.  
2. Select **Other** to review the remaining messages.  
3. Check both views regularly.

## Page 33 — Create a simple rule from a message  
## Create a simple email rule  
## Use this when  
Automatically handle predictable messages from a sender or with a consistent subject.  
## Good uses for simple rules  
* Move automated reports  
* Categorize system notifications  
* Move routine messages from a known sender  
* Flag messages containing a specific subject phrase  
* Separate lower-priority informational email  
## Create a rule from a message  
1. Select a message that represents the type of email you want the rule to handle.  
2. On the **Home** tab, select **Rules**.  
3. Select **Create Rule**.  
4. Choose one or more conditions, such as:  
    * From the sender  
    * Subject contains specific words  
    * Sent to a specific address  
5. Choose an action, such as:  
    * Display an alert  
    * Play a sound  
    * Move the item to a folder  
6. Select the destination folder when required.  
7. Select **OK**.  
8. Choose whether to run the rule on messages already in the current folder.  
9. Select **OK**.

After creating it:  
1. Send or locate a test message that matches the conditions.

3. Open the destination folder.

## What to remember  
Automate predictable email, not email that requires judgment.

* Do not create a rule without testing it.  
* Do not automatically move urgent or customer-facing messages unless the destination is monitored.  
* Do not use vague conditions that may catch unrelated messages.  
* Do not create several rules that perform conflicting actions.

## Page 35 — Manage and troubleshoot rules  
## Manage existing rules  
## Open Rules and Alerts  
1. Select **File**.  
2. Select **Manage Rules & Alerts**.  
3. Confirm the correct account.  
## Turn a rule on or off  
1. Locate the rule.  
2. Select the checkbox to turn it on.  
3. Clear the checkbox to turn it off.  
4. Select **Apply**.  
Turning a rule off preserves it for later use.  
## Edit a rule  
1. Select the rule.  
2. Select **Change Rule**.  
3. Choose:  
    * Edit Rule Settings  
    * Rename Rule  
    * Change an action  
4. Make the changes.  
5. Finish the wizard.  
6. Select **Apply**.  
## Change rule order  
1. Select the rule.  
2. Use the up or down controls.  
3. Review how the order affects other rules.  
4. Select **Apply**.  
Rule order matters when more than one rule matches the same message.  
## Run rules on existing messages  
1. Select **Run Rules Now**.  
2. Select the rule.  
3. Choose the folder.  
4. Select **Run Now**.  
## Delete a rule  
1. Select the rule.  
2. Select **Delete**.  
3. Confirm the deletion.  
4. Select **Apply**.  
## Troubleshooting table  

| Problem | Check |
| ------------------------------- | ------------------------------------------------------- |
| Rule does not run | Confirm the rule is turned on |
| Wrong messages move | Review conditions and exceptions |
| Message is processed twice | Review overlapping rules |
| Rule stops too early | Review rule order and stop-processing behavior |
| Folder is missing | Confirm the destination still exists |
| Rule works only on the computer | Review whether it is a client-only rule |
| New messages are missed | Test with a message that exactly matches the conditions |
  
****What to remember****  
Review rules when your work, mailbox or folder structure changes.  
## Avoid this  
* Do not delete a rule before recording what it does.  
* Do not leave broken rules active.  
* Do not assume rule order is unimportant.  
* Do not build rules that hide messages from normal review.  
## Platform difference  
Rules differ significantly across Classic Outlook, New Outlook, Outlook on the web and iPadOS. Some Classic Outlook rule actions are not available elsewhere.

Check:  
* The selected mailbox  
* The selected folder

5. Select **OK**.
