## High-level overview
Use this guide to complete common email, follow-up, organization, shared mailbox, and Microsoft 365 Copilot tasks in Classic Outlook for Windows. The guide follows one workflow: **Email → Decide → Make visible → Follow through**. When an email arrives, ask: **Does this need action from me?** If yes, ask: **What is the next step?** and **Where will I make it visible?**

## Step-by-step setup
1. Open Classic Outlook for Windows with your company-managed Microsoft 365 work account.
2. Confirm you are in Classic Outlook by looking for the traditional ribbon across the top.
3. Review the layout: folder pane, message list, reading pane, ribbon, search box, calendar, and shared mailboxes if available.
4. Set up your basic follow-up system: flags for action, categories for type of work, folders for reference, archive for completed email, and calendar items for time-bound follow-up.
5. Use Copilot only where it appears in your work account and review all output before sending.

## Page 6 — Take control workflow  
## Take control one decision at a time  
Most inbox challenges are not only about volume. They come from not knowing what still needs attention.  
Use this workflow whenever you open an email:  
## Email → Decide → Make visible → Follow through  
## Start with one question  
**Does this need action from me?**  
If the answer is yes, ask:  
1. **What is the next step?**  
2. **Where will I make it visible?**  
## Four doors for every email  

| Door | Use it when | What to do |
| ---------------------- | ---------------------------------- | -------------------------------- |
| Respond now | A quick reply resolves the message | Send the reply |
| Do the task | The work can be completed quickly | Complete it, then file the email |
| Make follow-up visible | More work or waiting is required | Flag, categorize or calendar it |
| Let it go | No action is required | File, archive or delete it |
  
****What to remember****  
* **Visible beats remembered.**  
* Make the next action easy to find.  
* Use a simple system you will notice later.  
* Copilot can help you move faster.  
* You remain responsible for the final message.  
**Copilot starts the draft. You finish the message.**

## How to recognize Classic Outlook  
Classic Outlook normally includes:  
* A **File** tab in the upper-left corner  
* A traditional command **ribbon**  
* A folder pane on the left  
* A message list in the center  
* A reading pane on the right or bottom  
* Mail and Calendar navigation  
* A **Send/Receive** tab on the ribbon  
Microsoft identifies the presence of a Send/Receive tab as one way to distinguish Classic Outlook from New Outlook or Outlook on the web. ## Page 14 — Use Reply All or Forward  
## Use Reply All or Forward  
## Reply to everyone  
Use **Reply All** only when everyone included in the message needs your response.  
1. Open or select the email.  
2. Select **Reply All**.  
3. Review the names in the **To** and **Cc** fields.  
4. Remove anyone who does not need the response.  
5. Write your message.  
6. Review the message and recipients.  
7. Select **Send**.  
## Forward a message  
Use **Forward** when someone who was not part of the original conversation needs the information.  
1. Open or select the email.  
2. Select **Forward**.  
3. Add the new recipient.  
4. Write a short explanation above the forwarded message.  
5. Review the original message and attachments.  
6. Remove information the new recipient does not need, when appropriate.  
7. Select **Send**.  
When you forward an email, its original attachments are normally included. Attachments are not automatically included when replying.  
## Add context when forwarding  
Do not send a forwarded message without explaining why the recipient is receiving it.  
Use a brief introduction such as:  
Please review the request below and let me know whether you can provide the missing information by Thursday.  
## Pop out a response  
Use a separate message window when you need more room to work.  
1. Start the reply or forward.  
2. Select **Pop Out**.  
3. Complete the response in the new window.  
4. Review and send it.  
## Decision guide  

| Choose    | Use when                                |
| --------- | --------------------------------------- |
| Reply     | Only the sender needs your response     |
| Reply All | Everyone included needs the response    |
| Forward   | A new person needs the original context |
  
****Avoid this****  
* Do not use Reply All by habit.  
* Do not forward customer or business information to someone who does not need it.  
* Do not assume that every original attachment should be forwarded.  
* Do not add a new topic to an unrelated email thread.  
## Platform difference  
New Outlook, Outlook on the web and Outlook on iPadOS place the response commands in different locations. The decision between Reply, Reply All and Forward remains the same.

## Page 15 — Create a new email  
## Create a new email  
## Use this when  
Start a new conversation that does not belong in an existing email thread.  
## Start the message  
1. In Mail, select the **Home** tab.  
2. Select **New Email**.  
3. Enter the primary recipients in the **To** field.  
4. Add optional recipients in the **Cc** field.  
5. Enter a clear subject.  
6. Write the message.  
## Use the recipient fields correctly  

| Field | Use |
| ----- | ------------------------------------------------------------------------------------------ |
| To | People expected to respond or take action |
| Cc | People who need the information but are not responsible for the action |
| Bcc | People who should receive the message without displaying their address to other recipients |
  
Use Bcc only when there is a clear business reason.  
## Write a useful subject  
The subject should help the recipient understand the topic and required action.  
## Less useful  
Follow-up  
## More useful  
Action needed: Confirm July 1 effective date by Friday  
## Less useful  
Question  
## More useful  
Question about Q3 renewal proposal assumptions  
## Structure the message  
Use this simple order:  
1. Purpose  
2. Requested action  
3. Deadline  
4. Supporting details  
5. Closing  
## Example  
**Subject:** Action needed: Confirm renewal effective date by June 20  
Hi Jordan,  
Please confirm the effective date for the renewal request below by June 20.  
We currently have July 1 listed, but the producer’s message references August 1. Once the date is confirmed, I can complete the update.  
Thank you,
[Name]

## Page 16 — Review, save or discard a message  
## Review before sending  
Before selecting **Send**, check:  
* Are the recipients correct?  
* Does everyone included need the message?  
* Is the subject clear?  
* Is the requested action easy to find?  
* Is the deadline included?  
* Are names, dates and account details accurate?  
* Are the correct files or links attached?  
* Does the tone fit the recipient?  
* Does the message contain only necessary information?  
## Send the message  
1. Complete the final review.  
2. Select **Send**.  
3. Confirm that the message window closes.  
4. Check **Sent Items** when you need to confirm that the message was sent.  
## Save a draft  
Outlook normally saves unfinished messages in the **Drafts** folder.  
To save and return later:  
1. Begin creating or replying to a message.  
2. Select **Save** on the Quick Access Toolbar or press **Ctrl+S**.  
3. Close the message window.  
4. Open the **Drafts** folder when you are ready to continue.  
5. Double-click the draft.  
6. Finish and send the message.  
Depending on the Outlook configuration, closing an unfinished message may display a prompt asking whether you want to save it.  
## Delete a draft  
1. Open the **Drafts** folder.  
2. Select the draft.  
3. Select **Delete**.  
Or:  
1. Open the draft.  
2. Select **Delete** or close the message without saving.  
3. Confirm the action when prompted.  
## What to remember  
A draft is unfinished work. Return to it, make the next action visible or delete it when it is no longer needed.  
## Avoid this  
* Do not leave important messages in Drafts without a follow-up plan.  
* Do not assume that a message was sent because it no longer appears on the screen.  
* Do not send before checking recipients and attachments.  
* Do not use an old thread when a new subject is needed.

## Attach a file  
## Use this when  
Send a fixed copy of a file with an email.  
## Before you start  
Confirm:  
* The file is the correct version.  
* The recipient is allowed to receive it.  
* The file does not contain unnecessary sensitive information.  
* A cloud link would not be more appropriate.

2. On the **Message** tab, locate the **Include** group.

4. Choose a recent file or select **Browse This PC**.  
5. Locate the file.  
6. Select the file.  
7. Select **Insert**.  
8. Confirm that the file name appears in the message.  
The Include group may appear only after you start a new message, reply or forward.  
## Drag a file into a message  
When enabled:  
1. Open File Explorer.  
2. Locate the file.  
3. Drag it into the open message.  
4. Confirm that it appears as an attachment.  
## Remove an attachment before sending  
1. Locate the attachment in the message.  
2. Select the attachment or its menu.  
3. Select **Remove Attachment**.  
4. Confirm that the file no longer appears.

## Build a clear prompt  
A clear prompt has four parts.  

| Part        | Question                                 |
| ----------- | ---------------------------------------- |
| Goal        | What do you want Copilot to do?          |
| Context     | What background matters?                 |
| Source      | What email or information should it use? |

****Prompt formula****  
**[Goal]** using **[source and context]**. Make it **[tone, length and format]**. Include **[required information]**. Do not **[important restriction]**.  
## Example: Ask for missing information  
Draft a warm professional reply to this thread. Confirm that I received the request, ask for the missing effective date and keep the reply under 100 words. Do not add facts that are not in the thread.  
## Example: Confirm next steps  
Draft a reply confirming the next steps, owners and deadlines stated in this thread. Use bullets. If a detail is not stated, leave it out.  
## Example: Provide a status update  
Draft a concise status update using the information in this thread. Include completed work, open items and the next expected update. Keep it under 150 words.  
## Example: Clarify a request  
Draft a friendly reply asking the sender to confirm the requested coverage period and deadline. Keep the questions easy to answer.  
## Add useful constraints  
Tell Copilot to:  
* Keep the same facts  
* Do not add new information  
* Use bullets  
* Keep it under a word limit  
* Ask one clear question  
* Put the requested action first  
* Use a warm professional tone  
* Mark missing details as not specified

Better instructions improve the draft, but they do not guarantee accuracy.

* Do not ask Copilot to invent missing details.  
* Do not include unnecessary sensitive information in the prompt.  
* Do not assume Copilot knows business context that is not in the message.
