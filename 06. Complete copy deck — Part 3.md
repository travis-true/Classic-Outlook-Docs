## Page 19 — Decide what an email needs  
## Decide what an email needs  
## Use this when  
Use this process whenever an email arrives and you are unsure what should happen next.  
## Start with one question  
**Does this need action from me?**  
## If the answer is no  
Choose the appropriate action:  
* Leave it temporarily if you still need to read it  
* Move it to a reference folder  
* Archive it  
* Delete it when it has no continuing value  
## If the answer is yes  
Ask:  
1. **What is the next step?**  
2. **Who owns the next step?**  
3. **When does it need to happen?**  
4. **Where will I make it visible?**  
## Choose one of four doors  

| Door | Use when | Action |
| ---------------------- | --------------------------------------------- | -------------------------------------- |
| Respond now | A quick response completes your part | Reply and send |
| Do the task | The work is short and can be finished now | Complete the work, then file the email |
| Make follow-up visible | The work requires more time or another person | Flag, categorize, calendar or track it |
| Let it go | No action is required | File, archive or delete |
  
****A simple decision process****  
1. Read the message.  
2. Identify the requested action.  
3. Confirm whether the action belongs to you.  
4. Name the next physical step.  
5. Choose the follow-up method.  
6. Complete, track or file the message.  
## Example  
**Message:** A producer asks for confirmation of a renewal effective date.  
**Decision:** Action is required.  
**Decision:** Action is required.  
**Next step:** Confirm the effective date with the appropriate source.  
**Next step:** Confirm the effective date with the appropriate source.  
**Visible follow-up:** Flag the email for tomorrow and categorize it as Follow-up.  
**Visible follow-up:** Flag the email for tomorrow and categorize it as Follow-up.  
**Completion:** Reply when the date is confirmed, then mark the flag complete.  
**Completion:** Reply when the date is confirmed, then mark the flag complete.  
## What to remember  
“Handle this later” is not a clear next step. Name the action you will take.

## Page 20 — Choose a follow-up method  
## Choose a follow-up method  
Different types of work need different follow-up tools.  
## Use the method you will notice  

| Method | Best use | Example |
| ------------ | ------------------------------------------- | ---------------------------------------------- |
| Flag | Return to a specific email by a target date | Reply after receiving missing information |
| Reminder | Receive an alert at a specific time | Review a proposal tomorrow morning |
| Category | Group or identify related work | Mark messages as Customer, Producer or Waiting |
| Calendar cue | Reserve focused time to complete work | Review renewal documents for 30 minutes |
| Task | Track work with several steps | Prepare a detailed account update |
| Folder | Store completed or reference messages | Save finalized renewal correspondence |
  
****Decision guide****  
## Use a flag when  
* The email itself contains the information you need  
* You need to return to the message later  
* The task has a clear target date  
* You are waiting for a response  
## Use a category when  
* You need to group related messages  
* You want a visible work-status label  
* Several messages belong to the same customer, producer or work type  
## Use a calendar cue when  
* The work needs focused time  
* The task cannot be completed in a quick response  
* You need to protect time on your calendar  
* The work has a specific date or deadline  
## Use a folder when  
* The message no longer requires action  
* You may need the message as a reference  
* The message belongs with related completed work  
## Combine tools only when useful  
A message may use more than one tool.  
**Example:**  
**Example:**  
* Category: Waiting  
* Flag: Friday  
* Folder: Customer Renewals after completion  
Do not add tools that do not improve visibility.  
## What to remember  
A simple follow-up method you use consistently is better than a complex system you ignore.  
## Avoid this  
* Do not flag every message.  
* Do not move unfinished work into a folder without tracking the next step.  
* Do not rely on categories alone when a deadline requires a reminder.  
* Do not create duplicate reminders in several places unless each one serves a purpose.

## Page 21 — Flag a message for follow-up  
## Flag a message for follow-up  
## Use this when  
Return to a message later, track a due date or show that additional work is required.  
## Add a quick flag  
1. Find the message in the message list.  
2. Point to the flag column beside the message.  
3. Select the flag icon.  
4. Confirm that the flag appears.  
The default follow-up date depends on your Outlook settings.  
## Choose a follow-up date  
1. Right-click the flag beside the message.  
Or:  
1. Select the message.  
2. On the **Home** tab, select **Follow Up**.  
Then choose:  
* Today  
* Tomorrow  
* This Week  
* Next Week  
* No Date  
* Custom  
1. Select the date that matches when you intend to act.  
2. Confirm that the flag appears beside the message.  
## Use Custom follow-up  
1. Right-click the message.  
2. Select **Follow Up**.  
3. Select **Custom**.  
4. Choose the **Start date**.  
5. Choose the **Due date**.  
6. Add a reminder when needed.  
7. Select **OK**.

## View flagged messages  
Depending on the available Outlook layout, use one of these methods:  
## Use the To-Do Bar  
1. Select the **View** tab.  
2. Select **To-Do Bar**.  
3. Select **Tasks**.  
4. Review the flagged messages and tasks.  
## Use the For Follow Up Search Folder  
1. Look in **Search Folders** in the Folder Pane.  
2. Open **For Follow Up**.  
3. Review the flagged messages.  
If the folder does not exist:  
1. Select the **Folder** tab.  
2. Select **New Search Folder**.

## Mark a follow-up complete  
Use this when the work is finished.  
1. Right-click the flagged message.  
2. Select **Follow Up**.  
3. Select **Mark Complete**.  
The flag changes to a completed indicator and is removed from active follow-up views.  
## Clear a flag  
Use this when the flag is no longer needed but the work should not be recorded as completed.  
1. Right-click the flagged message.  
2. Select **Follow Up**.  
3. Select **Clear Flag**.  
## Complete or clear?  

| Action        | Use when                                                |
| ------------- | ------------------------------------------------------- |
| Mark Complete | The required work was finished                          |
| Clear Flag    | The flag was added incorrectly or is no longer relevant |
  
****Avoid this****  
* Do not dismiss a reminder without deciding the next action.  
* Do not clear a flag merely to remove it from view.  
* Do not leave completed work flagged.  
* Do not use the same due date for every message.  
## Platform difference

## Page 26 — Use calendar follow-up effectively  
## Use calendar follow-up effectively  
A calendar cue should protect enough time to complete the work.  
## Choose the right amount of time  

| Work type                  | Example time block        |
| -------------------------- | ------------------------- |
| Quick review               | 15 minutes                |
| Draft or detailed response | 30 minutes                |
| Account research           | 30–60 minutes             |
| Proposal comparison        | 45–60 minutes             |
| Complex preparation        | Break into several blocks |
  
Adjust the time to match the actual work.  
## Add useful appointment details  
Include:  
* The required outcome  
* The email subject or sender  
* The deadline  
* Important notes  
* A link or copied context when available  
* The person who needs the response  
## Example  
**Subject:** Review proposal differences and reply to Jordan  
**Notes:**  
**Notes:**  
* Compare current and proposed terms  
* Confirm effective date  
* Identify unanswered questions  
* Send response by Thursday at 3 p.m.  
## When to create a meeting instead  
Create a meeting only when people need to discuss, decide or coordinate together.  
Use an appointment when:  
* You need personal focus time  
* No other attendee is required  
* The task can be completed independently  
Use a meeting when:  
* A decision requires several people  
* Participants must coordinate availability  
* Discussion is more efficient than an email exchange  
## Complete the loop  
After finishing the calendar work:  
1. Return to the email.  
2. Send the response or complete the required task.  
3. Mark the flag complete when one exists.  
4. Update or remove the category when appropriate.  
5. Move, archive or file the email.  
## What to remember  
Reserving time is not the same as completing the task. Return to the email and close the loop.  
## Avoid this  
* Do not schedule more time than the task requires.  
* Do not create a meeting when focused work or a clear email will resolve the issue.  
* Do not leave the original email flagged after the work is complete.  
* Do not create vague appointments such as “Email follow-up.”  
* Do not place unnecessary sensitive details in the calendar entry.  
## Platform difference  
Creating calendar items from email works differently in New Outlook, Outlook on the web and Outlook on iPadOS. Use the platform-specific guide for the exact workflow.
